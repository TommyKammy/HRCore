## Missing or Ambiguous Evidence Links

- MVP-A initial implementation issue scope is canonical as issues #175-#182,
  while the merged PR sequence includes PRs #175-#177 and #179-#183. This
  inventory keeps that non-contiguous PR range explicit instead of inferring a
  missing PR #178.
- MVP-D is accepted as bounded evidence hardening for CSV/Ops/DLQ, not a full
  employee-lifecycle E2E flow. Its matrix entries therefore mark request,
  approval, apply, and writeback surfaces only where the CSV/Ops/DLQ closeouts
  directly provide evidence.
- Cross-flow traceability is anchored to directly linked evidence for each
  bounded flow. Same-parent, sibling, naming-convention, issue-text, branch-name,
  or nearby-metadata inference is not accepted as evidence linkage.
- Verification counts are historical per closeout unless the current branch is
  explicitly re-run. The current branch adds this focused inventory guard, so the
  compiled test inventory is expected to report 447 tests when the full test
  path runs after this document exists.
